# Sentiment Analysis Project

Проект содержит два способа работы с анализом тональности текста:
- API на FastAPI
- Web-интерфейс на Streamlit

## Структура проекта

- `main.py` - FastAPI приложение
- `services.py` - общая логика анализа текста
- `streamlit_app.py` - Web-интерфейс на Streamlit

## Запуск проекта

Установить зависимости:

```bash
pip install fastapi uvicorn streamlit transformers torch
```

Запуск FastAPI:

```bash
uvicorn main:app --reload
```

Запуск Streamlit:

```bash
streamlit run streamlit_app.py
```

## Доступные API endpoints

- `GET /` - проверка запуска сервиса
- `GET /health` - healthcheck
- `POST /predict/` - анализ текста

Пример тела запроса:

```json
{
  "text": "I like this project"
}
```

## Идеи по улучшению качества кода

При анализе проекта были выделены следующие улучшения:
- вынести бизнес-логику из FastAPI-маршрутов в отдельный сервисный слой;
- убрать слишком общий маршрут `/{text}`, который может конфликтовать с другими endpoint;
- переиспользовать общую логику между API и Web-интерфейсом;
- добавить проверку пустого текста перед анализом;
- сделать ответы API типизированными через Pydantic-модели;
- обновить документацию проекта.

Реализованные улучшения:
- бизнес-логика анализа вынесена в `services.py`;
- добавлены типизированные модели ответа FastAPI;
- добавлена базовая проверка пустого ввода в общей логике сервиса;
- подготовлена документация по запуску и структуре проекта.
